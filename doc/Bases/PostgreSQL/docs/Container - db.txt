             Observações sobre o container Docker/Postgres/pgAdmin
a) Estou mantendo um volume chamado './data/pgadmin' e './data/postgres'
na raiz do projeto 'assjur-api'; Isso faz com que os dados inseridos no
banco de dados do conteiner sobrevivam quando houve a interrupção ou 
mesmo a remoção do container e sua imagem;

b) a diretiva 'volumes:' inserida em cada serviço já faz a criação
das respectivas pastas, no host; quando estiver rodando na vps, ficará
na pasta onde copiado a configuração do container;

c) diretiva '--build' só deve ser utilizada para fins de debug na execução
dos containers;

d) o postgres e o pgadmin rodam no mesmo ambiente e se comunicam por meio
da rede local 'local-network', por isso identificam os HOST recíproco pelo
seu  'container_name'; ou seja, Para conectar ao PostgreSQL usando o pgAdmin, 
você deve usar pg-server como o host, conforme mencionado nos comentários. 
Isso se deve ao fato de que os containers estão na mesma rede e podem se 
comunicar pelo nome do serviço.

e) COMANDOS:
Uso:
Os comandos fornecidos nos comentários permitem iniciar e parar os serviços, 
além de construir as imagens e remover imagens não utilizadas. Aqui estão os 
comandos principais:

Para iniciar os serviços em segundo plano:

docker compose -f compose-postgres.yaml up -d
Para parar os serviços:

docker compose -f compose-postgres.yaml down
Para construir as imagens e iniciar os serviços:

docker compose -f compose-postgres.yaml up --build
Para parar os serviços e remover todas as imagens:

docker compose -f compose-postgres.yaml down --rmi all
